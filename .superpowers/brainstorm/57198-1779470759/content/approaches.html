Implementation approach

3 ways to go full Clean Sci-Fi — pick the one that fits

A

Editor Script — Recommended ✦

Write a StrafAdvance > 10. Apply Sci-Fi Upgrade menu item that creates all materials from code and assigns them to prefabs. One-click, reproducible, version-controlled. Runs in seconds via MCP execute_menu_item.

Pros

  • Fast to iterate
  • Survives prefab resets
  • Easy to extend

Cons

  • Limited to material properties — no custom shader nodes

Covers: all 4 areas (materials, HUD, post-processing, VFX) via separate sub-functions

B

Custom Shader Graph

Build custom URP Shader Graphs with fresnel rim glow, holographic scanline distortion, energy shield flicker. Maximum visual fidelity. Assigned via same editor script approach.

Pros

  • Unique look impossible with standard shaders
  • Fresnel edge glow on ships/enemies

Cons

  • Shader Graph setup is slow through MCP
  • Harder to debug

Best for: if you want a signature look beyond stock URP Lit

C

Hybrid — Script + Shader Graph

Use the editor script for 80% (materials, HUD, post-processing, VFX) and add one custom Shader Graph for the player ship only — fresnel rim + animated energy pulse. Best balance of speed and visual impact.

Pros

  • Full coverage in one session
  • Player hero shader stands out

Cons

  • Slightly more complex than pure script

Recommended if you want the absolute best result
